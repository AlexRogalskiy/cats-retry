Sleep

You can configure how the delays between retries are implemented.

This is done using the Sleep type class:

trait Sleep[M[_]] {

  def sleep(delay: FiniteDuration): M[Unit]

}

Out of the box, the core module provides instances for Id and Future that simply do a Thread.sleep(...).

import retry.Sleep
// import retry.Sleep

import cats.Id
// import cats.Id

import scala.concurrent.duration._
// import scala.concurrent.duration._

Sleep[Id].sleep(10.milliseconds)
// res0: cats.Id[Unit] = ()

import scala.concurrent.Future
// import scala.concurrent.Future

import scala.concurrent.ExecutionContext.Implicits.global
// import scala.concurrent.ExecutionContext.Implicits.global

Sleep[Future].sleep(10.milliseconds)
// res1: scala.concurrent.Future[Unit] = Future(<not completed>)

The cats-effect module provides an instance that uses a cats-effect Timer.

import cats.effect.IO
// import cats.effect.IO

import scala.concurrent.ExecutionContext.Implicits.global
// import scala.concurrent.ExecutionContext.Implicits.global

import retry.CatsEffect._
// import retry.CatsEffect._

Sleep[IO].sleep(10.milliseconds)
// res2: cats.effect.IO[Unit] = IO$1546207770

Being able to inject your own Sleep instance can be handy in tests, as you can mock it out to avoid slowing down your unit tests.
